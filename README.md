# Лабораторная работа №2: Разработка лексического анализатора (сканера)

## Цель работы
Изучить назначение и принципы работы лексического анализатора в структуре компилятора. Спроектировать алгоритм (диаграмму состояний) и выполнить программную реализацию сканера для выделения лексем из входного текста. Интегрировать разработанный модуль в ранее созданный графический интерфейс языкового процессора.

## Постановка задачи
Разработать лексический анализатор (сканер), который:
- принимает на вход строку (исходный текст программы на Java);
- выделяет все допустимые лексемы согласно варианту задания;
- классифицирует лексемы по типам (строковая константа, идентификатор, ключевое слово, оператор, разделитель);
- обнаруживает ошибки (незакрытые строки, недопустимые escape-последовательности, недопустимые символы) и выводит сообщение с указанием позиции;
- учитывает многострочность входного текста;
- интегрирован в графический интерфейс с возможностью навигации по ошибкам.


## Вариант задания
**Номер варианта:** 59 
Объявление и инициализация строковой константы на языке Java.

### Примеры корректных входных строк:
1. `String message = "Hello World";`
2. `String path = "C:\\Program Files\\Java";`
3. `String text = "Hello\nWorld\t!";`


### Перечень допустимых лексем:

| Код | Тип лексемы | Описание | Примеры |
|-----|-------------|----------|---------|
| 1 | строковая константа | Последовательность символов в кавычках | `"Hello"`, `"Test\n"` |
| 2 | целое без знака | Последовательность цифр | `123`, `42` |
| 3 | идентификатор | Имя переменной (буквы, цифры, _) | `message`, `path`, `x1` |
| 4 | ключевое слово | Служебное слово Java | `String`, `int`, `class` |
| 5 | оператор присваивания | Символ = | `=` |
| 6 | конец оператора | Символ ; | `;` |
| 7 | пробел | Пробельный символ | ` `, `\t` |
| 8 | оператор + | Сложение/конкатенация | `+` |
| 9 | оператор - | Вычитание | `-` |
| 10 | оператор / | Деление | `/` |
| 11 | оператор * | Умножение | `*` |
| 12 | открывающая скобка | Левая круглая скобка | `(` |
| 13 | закрывающая скобка | Правая круглая скобка | `)` |
| 14 | ошибка | Недопустимая лексема | любой недопустимый символ |

### Описание работы автомата:

1. **Начальное состояние START** - ожидание первого символа лексемы.
2. **Состояние 1 (IN_STRING)** - обработка строковой константы:
   - При открывающей кавычке `"` переход в состояние 1
   - Накопление символов строки до закрывающей кавычки
   - При встрече `\` переход в состояние обработки escape-последовательности
   - При встрече символа новой строки `\n` - генерация ошибки (незакрытая строка)
   - При встрече закрывающей кавычки `"` - формирование лексемы с кодом 1
3. **Состояние 2 (IN_NUMBER)** - обработка целого числа:
   - Накопление цифр
   - При встрече не-цифры - формирование лексемы с кодом 2
4. **Состояние 3 (IN_IDENT)** - обработка идентификатора:
   - Накопление букв, цифр и подчеркиваний
   - Проверка на совпадение с ключевым словом "String" (код 4)
   - Иначе - формирование лексемы с кодом 3
5. **Состояния 4-12** - обработка односимвольных лексем:
   - Каждый символ сразу формирует соответствующую лексему
6. **Состояние ERROR** - обработка ошибок:
   - При недопустимом символе - код 14
   - При незакрытой строке - код 14
   - При недопустимой escape-последовательности - код 14
